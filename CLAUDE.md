## Project overview

Single-page marketing website for "Pap√≠rnictv√≠ Hraƒçky Roztoky", a family-run stationery and toy shop in Roztoky, Czech Republic. The entire site is one static HTML file with no build step, no framework, and no package manager ‚Äî just `index.html` with inline `<style>` and `<script>` tags, plus a handful of self-hosted asset files (SVG illustrations, fonts, a couple of product photos).

## Commands

There is no build, lint, or test tooling. To preview the site locally:

```
python -m http.server 8080 --directory .
```

Then open `http://localhost:8080`. The file can also be opened directly in a browser (`file://`) since it has no server-side dependencies and fonts/images are all local (no CDN calls at runtime).

## Architecture

- `index.html` ‚Äî the entire site: markup, all CSS (in a single `<style>` block in `<head>`), and all JS (in a single `<script>` block before `</body>`). Sections are anchored (`#top`, `#novinky`, `#produkty`, `#o-nas`, `#recenze`, `#kontakt`) and correspond to the nav tabs in the header. For handover, a large Czech comment block sits between `<html>` and `<head>` mapping the whole project (where data lives, document structure, common edits, deploy set); the `<style>` opens with a table-of-contents comment and the main `<script>` with a numbered overview of its parts ‚Äî keep these in sync if you restructure.
- CSS custom properties for the color palette / spacing are defined once in `:root` at the top of the `<style>` block (`--paper`, `--ink`, `--kraft`, `--honey`, `--sage`, `--brick`, etc.) ‚Äî reuse these vars rather than hardcoding colors.
- Fonts (`Fraunces`, `Karla`, `Caveat`) are **self-hosted** via `@font-face` rules pointing at `fonts/*.woff2` (latin + latin-ext subsets, variable-font files so one file covers all weights). They used to load from Google's CDN via `@import`, but that transmits visitors' IP addresses to Google on every page view ‚Äî a real GDPR issue for a site that otherwise collects zero data ‚Äî so don't reintroduce the `@import`.
- Responsive layout collapses to single-column via one `@media (max-width: 860px)` block at the end of the stylesheet. That same block also switches the header nav to wrap onto its own row (`.header-inner{flex-wrap:wrap}` + `nav.tabs{width:100%}`) ‚Äî without it, the non-shrinking brand name squeezes `nav.tabs` down to a few px on narrow screens, so don't remove those rules without re-testing mobile widths. The same block also deliberately **lightens the phone layout**: it hides desktop-only extras (`.hero-art` = the hero logo column, `.onas-art-wrap`, `.section-vite`, payment-card icons, `.map-embed`) and the whole `#produkty`/`#o-nas`/`#recenze` sections **plus their nav tabs** (so no menu links point at hidden sections). A `.jen-desktop` / `.jen-mobil` utility pair (same 860px breakpoint) lets any block show different content on phone vs desktop ‚Äî used e.g. for a short mobile version of the textbook-cover guide in `#skolni-balicky`. A tiny JS helper (`zviditelniTab`, in the active-tab observer) horizontally scrolls the mobile nav so the active tab stays visible.
- A simple click-to-zoom lightbox (`#lightbox`, wired up at the bottom of the `<script>` block) applies to any `<img class="note-img">` or `<img class="note-art">` in the Novinky section.
- **Novinky cards are data-driven**: `.notes-grid` is empty in the HTML and populated at runtime by `renderNovinky()` (top of the `<script>` block) from the `window.NOVINKY` array in `data/novinky.js`. The owner edits only that data file (heavily commented in Czech) to add/change/remove news. It's a plain `<script src>`-loaded `.js` (not JSON+fetch) on purpose so it also works via `file://`. Render runs before the reveal/lightbox observers so they pick up the generated cards. Two card shapes: with `nadpis` ‚Üí tag+image+title+text card; without ‚Üí image-only card (gets `.note-plain`). An item with `rezervace: true` also gets a `.note-reserve` "Rezervovat k vyzvednut√≠" button whose `mailto:` is built in JS (encodeURIComponent) with the product title pre-filled ‚Äî click&collect reservations, no form/backend/GDPR surface. `obrazek` may be one path (string) or up to 3 (array) ‚Äî multiple render as a `.note-gallery` thumbnail row, each opening the lightbox. Optional `zobrazitOd`/`zobrazitDo` ("YYYY-MM-DD") make a card auto-appear/auto-hide by date (`novinkaViditelna()` filters before render, last day inclusive). Note this makes Novinky content JS-dependent (core shop info stays static + JSON-LD, so SEO is fine).
- **"Vƒõdƒõli jste?" tips banner** (`.section-vite`, between Produkty and O n√°s) is data-driven like Novinky: an IIFE reads the `window.TIPY` string array from `data/tipy.js` (again `<script src>`, not fetch, so `file://` works), shows one **random** tip on load, and advances to the next only when the visitor clicks "Dal≈°√≠ tip" (gentle fade, respects `prefers-reduced-motion`). If `data/tipy.js` is missing/empty the banner hides itself. Edit tips in that file (Czech guide inside) or via `editor.html`.
- **Opening-status indicator** (`#stav-obchodu` in the `.chalkboard` in `#kontakt`) is rendered by an IIFE at the end of `<script>` that fills every element with class `.js-stav` ‚Äî currently just this one. (A decorative "hanging wooden sign" copy of the status used to live in the hero; it and the hero shelf illustration were removed, so the status now lives only in Kontakt. The `.js-stav`/multi-element wiring stays, so a second copy could be re-added by giving any element that class.) It computes "Pr√°vƒõ otev≈ôeno ¬∑ do HH:MM" / "Zav≈ôeno ¬∑ otev√≠r√°me ‚Ä¶" from the current **Prague** time (`toLocaleString` timeZone), honoring the lunch break (Po‚ÄìP√° 9‚Äì12, 12:30‚Äì18; So 9‚Äì12; Ne closed) and Czech public holidays incl. movable Easter (Good Friday + Easter Monday via `velikonocniNedele`), refreshing each minute. A **`dovolena` constant** (`{od, do}` as YYYY-MM-DD, both days inclusive, or `null`) at the top of the IIFE overrides everything for vacation closures and shows "Dovolen√° ¬∑ otev√≠r√°me ‚Ä¶" ‚Äî set it back to `null` once the vacation ends. **Opening hours live in three places that must stay in sync**: the visible `.hours-row` list in `#kontakt` (the Po‚ÄìP√° row stacks its two time ranges via `.hours-day`/`.hours-time`), `intervalyDne()` in the JS, and the `openingHoursSpecification` JSON-LD in `<head>`.
- **Hero** (`#top`): left column is the text + CTAs, right column (`.hero-art`) now holds just the pencil logo (`.brand-logo`), vertically centered. It previously held the shelf illustration + the hanging status sign; both were removed and the section was made more compact. Clicking the logo / "Dom≈Ø" (any `a[href="#top"]`) or loading a `#top` URL scrolls to the very top via JS (`naVrsek`) ‚Äî needed because the sticky header (which wraps to two rows on mobile) is taller than a plain anchor jump accounts for.
- `editor.html` (repo root, standalone, opened via double-click) is a **foolproof form-based editor** for the owner to edit both **Novinky and Tipy** without touching JS syntax. It loads current `data/novinky.js` and `data/tipy.js` via `<script src>`. The Novinky section renders a form per item (add/delete/reorder/type toggle/rezervace checkbox, auto-fills image dimensions, shows thumbnail + warnings) and generates a valid `novinky.js`; the Tipy section (below it) is a simple add/delete/reorder list of tip texts that generates a valid `tipy.js` (each string apostrophe-wrapped with escaping, so any inner quotes/apostrophes are safe). Each section has its own "download" + "copy" buttons. Owner then moves the downloaded file into `data/`, adds any new images to `images/produkty/`, and runs the .bat. No backend ‚Äî it only downloads files locally. If you change the on-disk format of either data file, update the header/guide text that the editor emits so a regenerated file still matches.
- `#skolni-balicky` (nav tab "≈Ýkola", placed right after Novinky) holds two things: a **seasonal** school-supply-packages banner (`.skolni-card`) and a `.skolni-guide` how-to on choosing textbook covers. No backend anywhere ‚Äî contact is `mailto:` (pre-filled subject + body template) and `tel:`, deliberately no form (no GDPR surface). An HTML comment above the section explains how to hide it off-season (comment out the `<section>` and its "≈Ýkola" nav tab). Owner dislikes em-dashes in body copy ‚Äî use commas/periods.
- `#recenze` section is just a `.rating-banner` (aggregate 4,6/5 stars + link to Google) ‚Äî the individual quoted reviews were removed as stale/self-congratulatory. Don't re-add specific quotes without asking.
- `#kontakt` embeds an OpenStreetMap iframe (`.map-embed`) rather than Google Maps ‚Äî a Google Maps embed was unreliable/blocked in testing, and OSM avoids sending visitor data to Google (same reasoning as the self-hosted fonts). The iframe is **hidden on mobile** (redundant with the "Otev≈ô√≠t v map√°ch" button, keeps the phone page short); it still shows on desktop.
