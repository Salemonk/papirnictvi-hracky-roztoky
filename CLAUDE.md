# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

Single-page marketing website for "Papírnictví Hračky Roztoky", a family-run stationery and toy shop in Roztoky, Czech Republic. The entire site is one static HTML file with no build step, no framework, and no package manager — just `index.html` with inline `<style>` and `<script>` tags, plus a handful of self-hosted asset files (SVG illustrations, fonts, a couple of product photos).

## Commands

There is no build, lint, or test tooling. To preview the site locally:

```
python -m http.server 8080 --directory .
```

Then open `http://localhost:8080`. The file can also be opened directly in a browser (`file://`) since it has no server-side dependencies and fonts/images are all local (no CDN calls at runtime).

## Architecture

- `index.html` — the entire site: markup, all CSS (in a single `<style>` block in `<head>`), and all JS (in a single `<script>` block before `</body>`). Sections are anchored (`#top`, `#novinky`, `#produkty`, `#o-nas`, `#recenze`, `#kontakt`) and correspond to the nav tabs in the header. For handover, a large Czech comment block sits between `<html>` and `<head>` mapping the whole project (where data lives, document structure, common edits, deploy set); the `<style>` opens with a table-of-contents comment and the main `<script>` with a numbered overview of its parts — keep these in sync if you restructure.
- CSS custom properties for the color palette / spacing are defined once in `:root` at the top of the `<style>` block (`--paper`, `--ink`, `--kraft`, `--honey`, `--sage`, `--brick`, etc.) — reuse these vars rather than hardcoding colors.
- Fonts (`Fraunces`, `Karla`, `Caveat`) are **self-hosted** via `@font-face` rules pointing at `fonts/*.woff2` (latin + latin-ext subsets, variable-font files so one file covers all weights). They used to load from Google's CDN via `@import`, but that transmits visitors' IP addresses to Google on every page view — a real GDPR issue for a site that otherwise collects zero data — so don't reintroduce the `@import`.
- Responsive behavior is split into **two breakpoints** (deliberately — tablets get full content, phones get a lighter page):
  - **Layout, `@media (max-width: 860px)`**: collapses grids to single-column and switches the header nav to wrap onto its own row (`.header-inner{flex-wrap:wrap}` + `nav.tabs{width:100%}`) — without it, the non-shrinking brand name squeezes `nav.tabs` down to a few px on narrow screens, so don't remove those rules without re-testing mobile widths. No content is hidden here.
  - **Phone lightening, `@media (max-width: 700px)`**: hides decorative/less-important parts (`.hero-art` = the hero logo column, `.onas-art-wrap`, `.section-vite`, `.map-embed`) and the whole `#produkty`/`#o-nas`/`#recenze` sections **plus their nav tabs** (so no menu links point at hidden sections). Don't merge these back into one block — a 768px tablet must fall between them (single-column but full content). (A `.jen-desktop`/`.jen-mobil` utility pair for per-device text used to live here; it was removed once the last user became data-driven — re-add at this breakpoint if ever needed again.)
  - A tiny JS helper (`zviditelniTab`, in the active-tab observer) horizontally scrolls the nav so the active tab stays visible when the tab row overflows.
- A simple click-to-zoom lightbox (`#lightbox`, wired up at the bottom of the `<script>` block) applies to any `<img class="note-img">` or `<img class="note-art">` in the Novinky section.
- **Novinky cards are data-driven**: `.notes-grid` is empty in the HTML and populated at runtime by `renderNovinky()` (top of the `<script>` block) from the `window.NOVINKY` array in `data/novinky.js`. The owner edits only that data file (heavily commented in Czech) to add/change/remove news. It's a plain `<script src>`-loaded `.js` (not JSON+fetch) on purpose so it also works via `file://`. Render runs before the reveal/lightbox observers so they pick up the generated cards. Two card shapes: with `nadpis` → tag+image+title+text card; without → image-only card (gets `.note-plain`). An item with `rezervace: true` also gets a `.note-reserve` "Rezervovat k vyzvednutí" button whose `mailto:` is built in JS (encodeURIComponent) with the product title pre-filled — click&collect reservations, no form/backend/GDPR surface. `obrazek` may be one path (string) or up to 3 (array) — multiple render as a `.note-gallery` thumbnail row, each opening the lightbox. Optional `zobrazitOd`/`zobrazitDo` ("YYYY-MM-DD") make a card auto-appear/auto-hide by date (`novinkaViditelna()` filters before render, last day inclusive). Note this makes Novinky content JS-dependent (core shop info stays static + JSON-LD, so SEO is fine).
- **"Věděli jste?" tips banner** (`.section-vite`, between Produkty and O nás) is data-driven like Novinky: an IIFE reads the `window.TIPY` string array from `data/tipy.js` (again `<script src>`, not fetch, so `file://` works), shows one **random** tip on load, and advances to the next only when the visitor clicks "Další tip" (gentle fade, respects `prefers-reduced-motion`). If `data/tipy.js` is missing/empty the banner hides itself. Edit tips in that file (Czech guide inside) or via `editor.html`.
- **Opening-status indicator** (`#stav-obchodu` in the `.chalkboard` in `#kontakt`) is rendered by an IIFE at the end of `<script>` that fills every element with class `.js-stav` — currently just this one. (A decorative "hanging wooden sign" copy of the status used to live in the hero; it and the hero shelf illustration were removed, so the status now lives only in Kontakt. The `.js-stav`/multi-element wiring stays, so a second copy could be re-added by giving any element that class.) It computes "Právě otevřeno · do HH:MM" / "Zavřeno · otevíráme …" from the current **Prague** time (`toLocaleString` timeZone), honoring the lunch break (Po–Pá 9–12, 12:30–18; So 9–12; Ne closed) and Czech public holidays incl. movable Easter (Good Friday + Easter Monday via `velikonocniNedele`), refreshing each minute. A **`dovolena` constant** (`{od, do}` as YYYY-MM-DD, both days inclusive, or `null`) at the top of the IIFE overrides everything for vacation closures and shows "Dovolená · otevíráme …" — set it back to `null` once the vacation ends. **Opening hours live in three places that must stay in sync**: the visible `.hours-row` list in `#kontakt` (the Po–Pá row stacks its two time ranges via `.hours-day`/`.hours-time`), `intervalyDne()` in the JS, and the `openingHoursSpecification` JSON-LD in `<head>`.
- **Hero** (`#top`): left column is the text + CTAs, right column (`.hero-art`) now holds just the pencil logo (`.brand-logo`), vertically centered. It previously held the shelf illustration + the hanging status sign; both were removed and the section was made more compact. Clicking the logo / "Domů" (any `a[href="#top"]`) or loading a `#top` URL scrolls to the very top via JS (`naVrsek`) — needed because the sticky header (which wraps to two rows on mobile) is taller than a plain anchor jump accounts for.
- `editor.html` (repo root, standalone, opened via double-click) is a **foolproof form-based editor** for the owner to edit **Novinky, Tipy, and the seasonal section** (labelled "Sezóna" in the editor; internally still `skola`) without touching JS syntax. It loads `data/novinky.js`, `data/tipy.js`, and `data/skola.js` via `<script src>`. Structurally it's **three independent editors sharing one pattern** (data array → `render*` builds the form → `generuj*` builds a valid `.js` string → download/copy), with a sticky in-page nav (`.ed-nav`, links to `#ed-novinky`/`#ed-tipy`/`#ed-skola`) at the top; a header comment and a `<script>`-overview comment document this. The Novinky section renders a form per item (add/delete/reorder/type toggle/rezervace checkbox, auto-fills image dimensions, shows thumbnail + warnings) → valid `novinky.js`; the Tipy section is an add/delete/reorder list of tip texts → valid `tipy.js` (apostrophe-wrapped with escaping); the seasonal section has the section name, seasonal on/off toggle + date range, and an add/delete/reorder list of simple content cards (štítek/nadpis/text/obrázek, with a file picker reusing `zkusCesty`), where **each card can individually enable a "Zavolat"/"Napsat e-mail" button** (email adds subject/body fields that show only when that card's email toggle is on) → valid `skola.js` (generated via `JSON.stringify`, so any quotes/apostrophes are safe). Each section has its own "download" + "copy" buttons. Owner moves the downloaded file into `data/`, adds any new images to `images/produkty/`, and runs the .bat. No backend — downloads files locally. If you change a data file's on-disk format, update the header/guide the editor emits so a regenerated file still matches.
- `#skolni-balicky` is a **generic "season" slot** — a data-driven seasonal section whose whole content/name swaps per season (currently "Školní balíčky", could be Vánoce etc.). The `skola`/`SKOLA`/`skolni-balicky` names are **historical** (it began as a bespoke "school packages" layout that proved too rigid, then was generalized); user-facing copy calls it "Sezónní sekce" / "Sezóna". It's built by `renderSkola()` (in the `<script>`) from `window.SKOLA` in `data/skola.js` — same `<script src>` (not fetch) approach as Novinky/Tipy. Its nav tab sits right after Novinky and shows `SKOLA.nazev`. The `<section>` is an empty `.container` in the HTML; render fills it with a `.section-head` (`SKOLA.nazev`, which **also overwrites the nav tab's text** so one field renames both), a `.notes-grid` of **content cards reusing the shared `vytvorNoteCard()`** (the same sticky-note cards as Novinky — so its items use the Novinky card schema: štítek/nadpis/text/obrázek), . `skolaViditelna()` gates the section: `SKOLA.zobrazit` (on/off) plus optional `zobrazitOd`/`zobrazitDo` ("YYYY-MM-DD", inclusive) — when hidden it also `display:none`s the nav tab so no menu link points at a hidden section. **Contact buttons are per-card, not per-section**: any card can set `telefonTlacitko`/`emailTlacitko` and (for email) `emailPredmet`/`emailText`; `vytvorNoteCard()` renders a `.note-cta`/`.note-btn` row inside that card (email `mailto:` built via `encodeURIComponent`, tel number fixed). Novinky items don't set those flags, so they get no buttons. `vytvorNoteCard()` (extracted from `renderNovinky`) is the single card builder shared by both sections; renderSkola runs before the reveal/active-tab observers so generated `.reveal` cards are picked up (it's meant to run once at load — re-running orphans the reveal wiring). None of the old bespoke school CSS remains (`.skolni-card`, `.skolni-guide`, `.skolni-cta`, `.btn-onSage`, …) — the card buttons use `.note-cta`/`.note-btn` near the `.note` styles. No backend, deliberately no form (no GDPR surface). Owner dislikes em-dashes in body copy — use commas/periods.
- `#recenze` section is just a `.rating-banner` (aggregate 4,6/5 stars + link to Google) — the individual quoted reviews were removed as stale/self-congratulatory. Don't re-add specific quotes without asking.
- `#kontakt` embeds an OpenStreetMap iframe (`.map-embed`) rather than Google Maps — a Google Maps embed was unreliable/blocked in testing, and OSM avoids sending visitor data to Google (same reasoning as the self-hosted fonts). The iframe is **hidden on mobile** (redundant with the "Otevřít v mapách" button, keeps the phone page short); it still shows on desktop.

### Illustrations and images

- `images/policka-cela.svg` — the big two-shelf illustration that *used to* be in the hero. **No longer referenced by `index.html`** (the hero now shows the logo), but the file is kept in case it's wanted again. It's a hand-composed scene built from the individual component SVGs also present in `images/` (`kelimek-s-tuzkami.svg`, `knihy.svg`, `rubikova-kostka*.svg`, `vlacek.svg`, `medvidek.svg`, `pocitadlo.svg`, `kostky*.svg`) — those standalone files exist so the user can edit each piece in Inkscape. If a component is edited standalone, the matching embedded copy inside `policka-cela.svg` needs updating too — they're not linked automatically. Since none of these are referenced anymore, they can be deleted if you want to trim the repo (they don't ship anything to visitors either way).
- `images/vyloha.svg` — storefront illustration in the "O nás" section. Also edited in Inkscape by the user; the door window has a clip-path showing a shrunk copy of the shelf scene through the glass.
- `images/dovolena.svg` — the text-free "vacation hours" note card art (sun umbrella, ball, suitcase).
- `images/produkty/` — real product photos (JPG/PNG) used in Novinky cards.
- `images/platby/` — accepted payment method icons (Visa, Mastercard, Amex, Maestro, V Pay, Apple/Google/Samsung Pay) shown in the footer. Sourced from `github.com/datatrans/payment-logos` (CC-BY-SA-4.0, see `images/platby/CREDITS.txt`).
- `assets/logo.svg` — full logo (pencil mascot + wordmark), used at large size in the hero.
- `assets/logo-icon.svg` — mascot only, no text, tightly cropped; used as the small round icon in the header/footer brand (rendered on a white circle via CSS).
- `assets/favicon.svg` — same mascot icon, baked onto a white rounded-square background, used as the page favicon.
